Buttons

Buttons make common actions immediately visible and easy to perform with one click or tap.

TYPES

  • Global Add: Used to add content to canopy
  • Primary buttons: Primary buttons should be the single most important action on the page.
  • Secondary buttons: Secondary buttons should be used for common actions when a primary button is present or can be used for a primary action when less emphasis is needed in the view.
  • Tertiary buttons: A tertiary button can be used for multiple actions in a view when primary and secondary actions are already taken or when an action is needed that has low emphasis.
  • Flat buttons: Flat buttons can be used as an alternative to primary or secondary buttons when needed for design aesthetic purposes or when primary and secondary buttons are both present.
  • Button with icon: An icon can be added to any button to create more affordance to the action.
  • Icon button: An icon button is used when no text is needed for the action.

GUIDELINES

  • Placement
    • In a page view, overlay or preview, buttons should be placed in the upper right hand corner
    • Wrong: Left hand modal/dialog/drawer placement
    • In a modal or drawer buttons should be placed in the bottom left hand corner
    • Wrong: Right hand modal/dialog/drawer placement
    • In cards, buttons can be placed upper right or lower left depending on the use case.
    • Buttons should be on cards
    • Wrong: Floating on canvas
  • Sizing
    • Primary and Secondary buttons:
    • Corner radius: 5px Height: 32px Horizontal padding: 24px Horizontal margin: 24px Text: 16px, semibold 600 Width: Auto
    • Flat and Tertiary:
    • Height: 32px Horizontal padding: 12px Horizontal margin: 24px Corner radius: 5px Width: Auto Text: 16px, semibold 600
    • Icon buttons:
    • Height: 24px 5px border radius. Width: 24px
  • Content
    • Human and relatable
    • Clear and concise
    • Convey the actions expected (Add or add contact = good New = bad)
    • Button labels should be in Title Case, except for the CANCEL button
    • No labels in all CAPS
    • Labels should not be more than 3 words
  • Type
    • Family: ASAP
    • Weight: 600
    • Size: 16px

    BEST PRACTICES

    • When using multiple buttons, the following combinations are acceptable. Tertiary buttons shouldn’t be combined.
      • Primary and Secondary, Primary and Flat, Secondary and Flat, Primary and Icon
      • Flat + icon? (add credit + kabob menu as an example)
    • Flat buttons are typically used for “cancel” action within modals.
    • Icon buttons are generally used for menus and editing actions.
    • Icon button combos: no margin, 16px margin, 24px margin.
    • Remember to consider whether an action would fit in the Global Add Button so that a user can create content from anywhere within the product
    • Wrong: never use links for actions, links are used for navigation
